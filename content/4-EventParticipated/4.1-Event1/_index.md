# Summary Report: FCAJ Community Day

### Event Objectives

* Learn how to use AI to support daily work and how to organize personal notes.
* Find out about Amazon Q (Amazon Quick) tool for quick Q&A.
* Learn about CloudFront and how to set up cost limits.
* Learn why AI (LLMs) sometimes gives different answers to the same question.
* Hear about the 36-hour hackathon experience from the LotusHack team.
* Listen to real-world stories and issues from people working in companies.

### Speakers

* **Tinh Truong** - Platform Engineer at GoTymeX
* **Anh Pham** - Cloud Consultant at G-AsiaPacific VietNam
* **Mr. Nguyen Tuan Thinh** - DevOps/Cloud Engineer at FCAJ
* **Mr. DucDao** - Solutions Architect at Cloud Kinetics
* **Ms. Thao Nguyen, Mai Nguyen, Uyen Le** - GenAI Engineers at VIB
* **Ms. Vy Lam** - Senior Business Systems Analyst at VPBank

### Key Highlights

#### Using AI and Building a Second Brain
* How to input enough context so that AI can understand and answer better.
* Simple note-taking tips to organize information for long-term study.

#### Discovering Amazon Q (Amazon Quick)
* An introduction to how the Amazon Q assistant works.
* A live demo showing how to use Amazon Q to search for database information.

#### Optimizing Amazon CloudFront
* How to use CloudFront to make websites load faster.
* How to set cost limits on AWS services to avoid unexpected bills.

#### Non-Determinism in LLM Settings
* Understanding why AI sometimes answers differently even with the same question.
* A quick demo showing AI answers under different settings.

#### The 36-Hour LotusHack Hackathon Experience
* Team UTMorpho sharing how they built a drag-and-drop UI generator in just 16 coding hours.
* Funny situations like running out of AI credits, changing ideas halfway, and working under time pressure.

#### Real Stories from Companies
* Common mistakes when setting up systems that books do not teach.
* Three important terms to remember: Security, Reliability, and Scalability.

### Key Takeaways

* Learned to write better prompts to ask AI for help in my Battleship game project.
* Learned how to set up cost alerts to manage my AWS account.
* Got a basic understanding of why we should use Multi-Agent AI instead of just one AI.
* Learned about teamwork and quick problem-solving from the speakers.

### Event Experience

This was the first workshop I attended in the FCAJ AWS internship. The talks were very easy to follow and interesting, especially the stories from team UTMorpho and Ms. Vy Lam. These real-world shares help me understand what working in the tech industry looks like.
